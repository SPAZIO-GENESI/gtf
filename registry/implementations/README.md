# implementations/ — Implementazioni (IMP)

Un file `IMP-<slug>.yaml` per ogni puntatore versionato a codice o
configurazione reale (repo + riferimento a file/riga/config) che realizza un
controllo. Validato da `schemas/implementation.schema.json`.

Popolato in fase **M1 — Import dell'esistente** (vedi ARCHITECTURE.md §12).
